## 🛠️ Supabase 配置步骤

### 重要：必须配置 Redirect URLs

为了让密码重置功能正常工作，**必须**在 Supabase Dashboard 中配置重定向 URL。

#### 1. 登录 Supabase Dashboard

访问：https://supabase.com/dashboard

#### 2. 选择项目并进入 Authentication 设置

1. 选择你的项目（如：question-bank）
2. 点击左侧菜单 "Authentication"
3. 找到 "URL Configuration" 部分

#### 3. 配置 Redirect URLs

在 "Redirect URLs" 部分填入以下信息：

**生产环境：**
```
Site URL: https://fallrain0905.top

Redirect URLs:
  • https://fallrain0905.top/auth/callback
  • https://fallrain0905.top/reset-password
```

**本地开发环境：**
```
Site URL: http://localhost:3000

Redirect URLs:
  • http://localhost:3000/auth/callback
  • http://localhost:3000/reset-password
```

### 配置说明

- **Site URL**：你的网站根域名
- **Redirect URLs**：需要包含所有可能的重定向目标
- `/auth/callback`：OAuth 认证回调
- `/reset-password`：密码重置回调

### ⚠️ 不配置的后果

如果不配置这些 URL，用户点击密码重置链接时会出现以下问题：

- ❌ 链接点击无效
- ❌ 自动跳转到默认错误页面
- ❌ 无法完成密码重置流程

## 📱 功能使用流程

### 用户端操作流程

#### 步骤 1：申请密码重置

1. 用户访问登录页面：`http://fallrain0905.top/login`
2. 点击"忘记密码？"按钮
3. 输入注册时使用的邮箱地址
4. 点击"发送密码重置邮件"按钮
5. 系统显示"密码重置邮件已发送！请检查您的邮箱"

#### 步骤 2：接收邮件

1. 用户检查邮箱收件箱（可能需要检查垃圾邮件）
2. 找到来自 Supabase 的密码重置邮件
3. 邮件标题通常为："Reset your password"
4. 点击邮件中的重置链接

#### 步骤 3：设置新密码

1. 用户自动跳转到：`http://fallrain0905.top/reset-password`
2. 页面显示"设置新密码"表单
3. 输入新密码（至少 6 位）
4. 再次输入新密码确认
5. 系统实时显示密码强度（弱/中/强）
6. 点击"确认修改密码"按钮

#### 步骤 4：完成重置

1. 密码更新成功后显示成功提示
2. 系统自动清理当前会话
3. 2 秒后自动跳转到登录页面
4. 用户使用新密码登录

## 🔐 安全特性

### 1. 会话验证

```typescript
// 检查用户是否有有效会话（通过邮件链接验证后自动获得）
const { data: { session } } = await supabase.auth.getSession();

if (session) {
  // 允许设置新密码
} else {
  // 显示"链接已失效"，防止直接访问重置页面
}
```

### 2. 密码强度检测

```typescript
const getPasswordStrength = (password: string) => {
  let strength = 0;
  if (password.length >= 8) strength += 1;
  if (/[a-z]/.test(password)) strength += 1;
  if (/[A-Z]/.test(password)) strength += 1;
  if (/[0-9]/.test(password)) strength += 1;
  if (/[^a-zA-Z0-9]/.test(password)) strength += 1;

  return strength <= 2 ? '弱' : strength <= 4 ? '中' : '强';
};
```

### 3. 双重确认

```typescript
// 验证两次密码输入是否一致
if (newPassword !== confirmPassword) {
  setError('两次输入的密码不一致');
  return false;
}
```

### 4. 输入验证

- 邮箱格式验证：`/^[^\s@]+@[^\s@]+\.[^\s@]+$/`
- 密码最小长度：6 位字符
- 防止提交空值
- 加载状态防止重复提交

## 📞 故障排查

### 常见问题

#### 1. 邮件未收到

**可能原因：**
- Supabase 邮件服务延迟
- 邮箱地址输入错误
- 邮件被误判为垃圾邮件

**解决方法：**
- 检查邮箱地址是否正确
- 查看垃圾邮件文件夹
- 等待 1-2 分钟后再检查
- 尝试重新发送

#### 2. 重置链接无效

**可能原因：**
- Supabase Redirect URLs 未配置
- 链接已过期（有效期为 1 小时）
- 已使用过一次重置链接

**解决方法：**
- 检查 Supabase Redirect URLs 配置
- 重新申请密码重置
- 确保在链接有效期内点击

#### 3. 无法修改密码

**可能原因：**
- 用户会话已过期
- 网络连接问题
- API 错误

**解决方法：**
- 刷新页面重新获取会话
- 检查网络连接
- 查看浏览器控制台错误信息

#### 4. 新密码无法登录

**可能原因：**
- 密码修改未成功
- 输入了错误的新密码
- 缓存问题

**解决方法：**
- 确认密码修改成功提示
- 清除浏览器缓存和 Cookie
- 尝试使用"忘记密码"功能重新设置
