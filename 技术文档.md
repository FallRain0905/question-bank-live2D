## 📖 项目概述

### 项目简介

学习共享平台是一个基于 Next.js 15 和 Supabase 的现代化在线学习社区系统，提供题目上传、笔记分享、班级协作、AI 助手等完整功能。

### 核心特性

- **题库管理**：支持文本、图片、PDF/Word 多格式题目上传
- **智能解析**：自动解析数学公式（KaTeX）、文档题目提取
- **笔记系统**：多格式笔记上传与分享
- **班级协作**：学习班级创建、成员管理、资源共享
- **AI 助手**：千问 AI 智能问答，支持图文对话
- **社区互动**：评论、点赞、关注、收藏系统
- **个性化界面**：6 种主题配色 + 自定义背景图片
- **管理后台**：公告、分类、标签、系统配置管理
- **权限控制**：基于 Supabase RLS 的细粒度权限管理

### 技术架构说明

#### 1. 前端层
- **框架**：Next.js 15 App Router
- **渲染模式**：Server Components + Client Components 混合
- **路由**：文件系统路由 (`app/` 目录)
- **状态管理**：React Hooks + Context API
- **样式**：Tailwind CSS + CSS Modules

#### 2. 网关层
- **反向代理**：Nginx
- **负载均衡**：支持多进程
- **SSL/HTTPS**：支持证书配置
- **静态资源**：Next.js 自动优化和服务
- **请求限制**：`client_max_body_size 50M`

#### 3. 应用层
- **运行时**：Node.js V8
- **框架**：Next.js 15
- **API 路由**：`app/api/` 目录下的 Route Handlers
- **中间件**：Next.js Middleware + 自定义拦截器
- **缓存**：Next.js 内置缓存 + Supabase 缓存

#### 4. 数据层
- **数据库**：PostgreSQL（Supabase 托管）
- **认证**：Supabase Auth (JWT + Session)
- **存储**：Supabase Storage + Nextcloud WebDAV
- **权限**：Row Level Security (RLS) + 角色策略

#### 5. 外部服务
- **AI 服务**：千问 API（通义千问模型）
- **云存储**：Nextcloud WebDAV API
- **全文搜索**：PostgreSQL GIN 索引 + `to_tsvector`

### RLS 权限策略

#### 权限层次

1. **超级管理员**：
   - 邮箱白名单：`['3283254551@qq.com']`
   - 权限：全部数据的读写删除

2. **管理员**：
   - `user_profiles.is_admin = true`
   - 权限：审核、管理标签、公告

3. **普通用户**：
   - 只能操作自己创建的内容
   - 可查看公开内容

4. **未登录用户**：
   - 只能访问登录/注册页面
   - 无数据访问权限

#### 关键 RLS 策略

```sql
-- 题目表策略
CREATE POLICY "任何人都可以查看题目"
    ON questions FOR SELECT
    USING (auth.role() = 'authenticated');

CREATE POLICY "只有创建者可以删除题目"
    ON questions FOR DELETE
    USING (auth.uid() = user_id);

-- 评论表策略
CREATE POLICY "登录用户可以创建评论"
    ON comments FOR INSERT
    WITH CHECK (auth.role() = 'authenticated');

-- 笔记表策略
CREATE POLICY "用户可以管理自己的笔记"
    ON notes FOR ALL
    USING (auth.uid() = user_id);
```

### 索引优化

| 索引 | 类型 | 字段 | 用途 |
|--------|------|------|------|
| `idx_questions_user_id` | B-tree | user_id | 按用户查询题目 |

## 🔌 API 接口文档

### 前端 API 路由

#### 1. 文件上传接口

##### `/api/upload-background`
- **方法**：POST
- **用途**：上传背景图片到 Nextcloud
- **请求**：`FormData { file: File, path: string }`
- **响应**：
```json
{
  "success": true,
  "url": "https://nextcloud.example.com/s/xxx/download"
}
```

##### `/api/upload-questions`
- **方法**：POST
- **用途**：上传题目到 Supabase Storage
- **请求**：
```json
{
  "question_text": "题目内容",
  "question_image": "base64...",
  "answer_text": "答案内容",
  "tags": ["数学", "函数"]
}
```

##### `/api/upload-notes`
- **方法**：POST
- **用途**：上传笔记文件
- **请求**：`FormData { file: File, title: string, tags: string[] }`
- **响应**：
```json
{
  "success": true,
  "note_id": "uuid",
  "file_url": "https://supabase.example.com/..."
}
```

#### 2. AI 助手接口

##### `/api/test-ai`
- **方法**：POST
- **用途**：测试千问 AI 连接
- **请求**：
```json
{
  "message": "用户问题",
  "images": ["base64..."]
}
```

##### `/app/ai` 页面
- **实时对话**：使用 WebSocket 或 Server-Sent Events
- **多会话**：支持创建多个对话会话
- **流式响应**：实时显示 AI 回复

#### 3. 用户管理接口

##### `/api/users/[id]`
- **方法**：GET/PUT
- **用途**：获取/更新用户信息
- **参数**：`id` (UUID)

##### `/api/me/follow`
- **方法**：POST/DELETE
- **用途**：关注/取消关注用户
- **请求**：`{ "user_id": "uuid" }`

##### `/api/me/favorites`
- **方法**：GET/POST/DELETE
- **用途**：收藏管理（题目、笔记）
- **多态类型**：`question`、`note`

## 🚀 部署指南

### 环境要求

#### 服务器配置

- **操作系统**：Ubuntu 20.04+ / CentOS 7+ / 其他 Linux 发行版
- **Node.js**：v18.x 或更高版本
- **内存**：最低 2GB，推荐 4GB+
- **磁盘空间**：至少 20GB 可用空间
- **网络**：公网 IP 或域名

#### 软件依赖

```bash
# Node.js 和 npm
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
apt-get install -y nodejs

# PM2（进程管理）
npm install -g pm2

# Nginx（如果未安装）
apt-get install -y nginx

# Git（版本控制）
apt-get install -y git
```

### 部署步骤

#### 1. 克隆代码仓库

```bash
# 使用 Gitee
git clone https://gitee.com/FallRain0905/question-bank.git

# 或使用 GitHub
git clone https://github.com/FallRain0905/question-bank.git

cd question-bank
```

#### 2. 安装依赖

```bash
npm install
```

#### 3. 配置环境变量

创建 `.env.local` 文件：

```env
# Supabase 配置
NEXT_PUBLIC_SUPABASE_URL=your-supabase-url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key

# Nextcloud 配置
NEXTCLOUD_URL=https://your-nextcloud.com
NEXTCLOUD_USER=your-username
NEXTCLOUD_PASSWORD=your-password
NEXTCLOUD_PUBLIC_URL=https://your-nextcloud.com

# 千问 AI 配置（可选）
QWEN_API_KEY=your-api-key
QWEN_MODEL=qwen-turbo
```

#### 4. 配置数据库

在 Supabase Dashboard 中按顺序执行 SQL 脚本：

1. `supabase/schema.sql` - 基础表结构
2. `supabase/notes_schema.sql` - 笔记表
3. `supabase/class_system.sql` - 班级表
4. `supabase/ultimate_fix.sql` - 完整 RLS 策略

#### 5. 构建生产版本

```bash
npm run build
```

#### 6. 配置 Nginx

创建 `/etc/nginx/conf.d/question-bank.conf`：

```nginx
server {
    listen 80;
    listen 8080;  # 移动端端口（可选）

    server_name fallrain0905.top;  # 替换为你的域名或 IP

    client_max_body_size 50M;  # 文件上传限制

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;

        # 超时设置
        proxy_connect_timeout 300s;
        proxy_send_timeout 300s;
        proxy_read_timeout 300s;
    }
}
```

重启 Nginx：

```bash
sudo nginx -t           # 测试配置
sudo systemctl reload nginx  # 重载配置
```

#### 7. 使用 PM2 启动应用

创建 `ecosystem.config.js`：

```javascript
module.exports = {
  apps: [{
    name: 'question-bank',
    script: 'npm',
    args: 'start',
    cwd: '/root/question-bank',
    env: {
      NODE_ENV: 'production',
      NODE_OPTIONS: '--max-old-space-size=4096'
    },
    instances: 1,
    autorestart: true,
    watch: false,
    max_memory_restart: '1G',
    error_file: './logs/app-error.log',
    out_file: './logs/app-out.log',
    log_date_format: 'YYYY-MM-DD HH:mm:ss Z'
  }]
};
```

启动应用：

```bash
# 创建日志目录
mkdir -p logs

# 启动应用
pm2 start ecosystem.config.js

# 查看状态
pm2 status

# 查看日志
pm2 logs question-bank

# 设置开机自启
pm2 startup
pm2 save
```

## 🐛 故障排查

### 常见问题

#### 1. 413 错误（Request Entity Too Large）

**症状**：上传文件时出现 413 错误

**原因**：Nginx 文件上传大小限制太低

**解决**：
```nginx
# 在 Nginx 配置中添加
client_max_body_size 50M;
```

#### 2. 502 Bad Gateway

**症状**：访问网站显示 502 错误

**原因**：Next.js 应用未运行或端口不匹配

**解决**：
```bash
# 检查应用是否运行
pm2 status

# 检查端口是否正确
netstat -tulpn | grep 3000

# 重启应用
pm2 restart question-bank
```

#### 3. 数据库连接失败

**症状**：Supabase 连接错误

**原因**：环境变量配置错误或网络问题

**解决**：
```bash
# 检查环境变量
cat .env.local

# 测试 Supabase 连接
curl -I https://your-project.supabase.co

# 重新生成 API Keys
# 在 Supabase Dashboard 中重新生成
```

#### 4. Next.js 构建失败

**症状**：`npm run build` 报错

**原因**：TypeScript 类型错误或依赖冲突

**解决**：
```bash
# 清除缓存
rm -rf .next
rm -rf node_modules

# 重新安装依赖
npm install

# 类型检查
npm run lint

# 构建项目
npm run build
```
