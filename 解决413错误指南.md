# 413 错误解决指南

## 问题分析
**错误：413 (Request Entity Too Large)**

这表示上传的文件（1.6MB）超过了服务器的请求大小限制。

## 解决方案

### 1. Next.js 配置（已更新）

✅ 已在 `next.config.ts` 中添加：
```typescript
api: {
  bodyParser: {
    sizeLimit: '50mb',
  },
}
```

### 2. 反向代理配置（重要）

如果你使用 Nginx 作为反向代理，需要修改配置：

```nginx
# 在 http 或 server 块中添加
client_max_body_size 50M;

# 示例配置
server {
    listen 80;
    server_name your-domain.com;

    client_max_body_size 50M;  # 增加上这行

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

**修改后重启 Nginx：**
```bash
sudo nginx -t        # 测试配置
sudo systemctl reload nginx
```

### 3. Node.js 启动配置

如果直接使用 Node.js 运行，设置环境变量：

```bash
# Linux/Mac
export NODE_OPTIONS="--max-old-space-size=4096"

# Windows
set NODE_OPTIONS=--max-old-space-size=4096

# 然后启动
npm start
```

### 4. PM2 配置

如果使用 PM2 运行，修改 `ecosystem.config.js`：

```javascript
module.exports = {
  apps: [{
    name: 'question-bank',
    script: 'npm',
    args: 'start',
    env: {
      NODE_OPTIONS: '--max-old-space-size=4096'
    },
    max_memory_restart: '1G'
  }]
};
```

然后重新启动：
```bash
pm2 restart question-bank
```

### 5. Nextcloud 配置

检查 Nextcloud 的上传限制：

1. 登录 Nextcloud 管理界面
2. 进入"设置" → "管理" → "文件处理"
3. 修改"最大上传文件大小"为至少 50MB

### 6. PM2 启动脚本

创建启动脚本 `start.sh`：

```bash
#!/bin/bash
export NODE_OPTIONS="--max-old-space-size=4096"
npm start
```

然后用 PM2 启动：
```bash
pm2 start start.sh --name "question-bank"
```

## 快速诊断

### 检查哪个限制导致问题

1. **浏览器开发者工具**：查看具体的错误响应头
2. **Nginx 日志**：`tail -f /var/log/nginx/error.log`
3. **PM2 日志**：`pm2 logs question-bank`

### 测试步骤

1. 修改相关配置
2. 重启相关服务
3. 清除浏览器缓存
4. 重新测试上传

## 常见问题

**Q: 修改后还是 413 错误？**
A: 检查是否修改了正确的配置文件，是否重启了相关服务

**Q: 修改后网站打不开？**
A: 检查配置文件语法是否正确，查看服务日志

**Q: 不知道用什么服务器？**
A: 检查进程：`ps aux | grep -E 'nginx|apache|node'`
