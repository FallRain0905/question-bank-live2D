## 问题描述

### 主要问题
用户可以在班级页面看到自己创建的待审核班级，但在超级管理员的管理页面 (`/admin/classes`) 中看不到待审核的班级申请列表。

| 创建班级 | ✅ 正常工作 |
| 查看待审核班级 | ✅ 正常工作 |
| 班级审核列表 | ❌ 无法显示申请 |

## 问题分析

### 1. 数据流程

**创建班级流程**:
```
用户创建班级
  ↓
前端插入 classes 表 (status='pending')
  ↓
触发器自动插入 class_members 表（创建者为成员，status='approved'）
  ↓
前端插入 class_approval_requests 表（status='pending'）
  ↓
管理员应该在 /admin/classes 看到申请
```

### 2. 可能的问题点

#### 2.1 RLS 策略配置问题

**class_approval_requests 表当前策略**:
```sql
-- SELECT 策略
CREATE POLICY "class_approval_requests_select"
    ON class_approval_requests FOR SELECT
    USING (
        is_super_admin()   -- 超级管理员可查看所有
        OR user_id = auth.uid()  -- 用户可查看自己的
    );
```

**可能问题**:
- `is_super_admin()` 函数可能返回 `false`（邮箱不匹配）
- 或者函数本身有问题（需要检查 `user_profiles.email` 字段是否存在）

#### 2.2 数据插入问题

创建班级时代码:
```javascript
await supabase
  .from('class_approval_requests')
  .insert({
    class_id: data.id,
    user_id: user.id,
    name: data.name,
    description: data.description || '',
    invite_code: data.invite_code,
    status: 'pending',
  });
```

**可能问题**:
- RLS 策略阻止了 INSERT 操作
- `class_approval_requests` 表可能缺少 INSERT 策略

#### 2.3 前端查询问题

管理员页面查询代码:
```javascript
const { data, error } = await supabase
  .from('class_approval_requests')
  .select('*')
  .order('requested_at', { ascending: false });
```

**可能问题**:
- RLS 策略返回空结果
- 或者查询条件不正确

## 疑似问题

### 1. `class_approval_requests` 表缺少 INSERT 策略

如果该表没有 INSERT 策略，且 RLS 已启用，那么所有 INSERT 操作都会被阻止。

**解决方案**: 添加以下策略
```sql
CREATE POLICY "class_approval_requests_insert"
    ON class_approval_requests FOR INSERT
    WITH CHECK (
        auth.uid() = user_id
        AND status = 'pending'
    );
```

### 2. `is_super_admin()` 函数依赖 `user_profiles.email`

如果 `user_profiles` 表的 `email` 字段没有正确同步，函数会返回 `false`。

**解决方案**: 确保 `user_profiles.email` 字段存在且数据正确
```sql
-- 检查字段是否存在
SELECT column_name, data_type
FROM information_schema.columns
WHERE table_name = 'user_profiles' AND column_name = 'email';

-- 如果不存在，添加字段
ALTER TABLE user_profiles ADD COLUMN email TEXT;

-- 同步邮箱
UPDATE user_profiles up
SET email = au.email
FROM auth.users au
WHERE up.id = au.id AND up.email IS NULL;
```

### 3. 触发器问题

如果 `class_approval_requests` 的插入是通过触发器自动完成的，而不是前端代码，那么前端代码可能没有生效。

**需要检查**: 是否应该改用触发器自动创建 `class_approval_requests` 记录？

## 建议的完整修复方案

### 方案 A：使用触发器自动创建审核申请

修改 `classes` 表的触发器，在创建班级时自动插入 `class_approval_requests` 记录：

```sql
CREATE OR REPLACE FUNCTION create_approval_request()
RETURNS TRIGGER AS $$
BEGIN
    -- 创建审核申请记录
    INSERT INTO class_approval_requests (class_id, user_id, name, description, invite_code, status)
    VALUES (
        NEW.id,
        NEW.creator_id,
        NEW.name,
        NEW.description,
        NEW.invite_code,
        'pending'
    );
    RETURN NEW;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;

CREATE TRIGGER on_class_create_approval_request
    AFTER INSERT ON classes
    FOR EACH ROW
    EXECUTE FUNCTION create_approval_request();
```

### 方案 B：确保完整的 RLS 策略

为 `class_approval_requests` 表创建完整的 CRUD 策略：

```sql
-- INSERT 策略
CREATE POLICY "class_approval_requests_insert"

    USING (is_super_admin())

    ON class_approval_requests FOR DELETE
    USING (is_super_admin());
```

## 附：完整的 RLS 策略配置参考

```sql
-- ============================================
-- 完整的班级系统 RLS 配置
-- ============================================

-- 辅助函数
CREATE OR REPLACE FUNCTION is_authenticated()
RETURNS BOOLEAN AS $$
BEGIN
    RETURN auth.uid() IS NOT NULL;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;

CREATE OR REPLACE FUNCTION is_super_admin()
RETURNS BOOLEAN AS $$
DECLARE
    user_email TEXT;
BEGIN
    SELECT email INTO user_email
    FROM auth.users
    WHERE id = auth.uid();
    RETURN user_email = 'YOUR_ADMIN_EMAIL';
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;

CREATE OR REPLACE FUNCTION is_class_member(class_id_param UUID, status_param TEXT DEFAULT 'approved')
RETURNS BOOLEAN AS $$
BEGIN
    RETURN EXISTS (
        SELECT 1 FROM class_members
        WHERE class_id = class_id_param
        AND user_id = auth.uid()
        AND status = status_param
    );
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;

CREATE OR REPLACE FUNCTION is_class_creator(class_id_param UUID)
RETURNS BOOLEAN AS $$
BEGIN
    RETURN EXISTS (
        SELECT 1 FROM classes
        WHERE id = class_id_param
        AND creator_id = auth.uid()
    );
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;

-- classes 表策略
ALTER TABLE classes ENABLE ROW LEVEL SECURITY;

DROP POLICY IF EXISTS "classes_insert" ON classes CASCADE;
DROP POLICY IF EXISTS "classes_select" ON classes CASCADE;
DROP POLICY IF EXISTS "classes_update" ON classes CASCADE;

CREATE POLICY "classes_insert"
    ON classes FOR INSERT
    WITH CHECK (
        is_authenticated()
        AND creator_id = auth.uid()
        AND status = 'pending'
    );

CREATE POLICY "classes_select"
    ON classes FOR SELECT
    USING (
        is_super_admin()
        OR creator_id = auth.uid()
        OR is_class_member(id, 'approved')
    );

CREATE POLICY "classes_update"
    ON classes FOR UPDATE
    USING (is_class_creator(id))
    WITH CHECK (is_class_creator(id));

-- class_members 表策略
ALTER TABLE class_members ENABLE ROW LEVEL SECURITY;

DROP POLICY IF EXISTS "class_members_select" ON class_members CASCADE;
DROP POLICY IF EXISTS "class_members_insert" ON class_members CASCADE;
DROP POLICY IF EXISTS "class_members_update" ON class_members CASCADE;
DROP POLICY IF EXISTS "class_members_delete" ON class_members CASCADE;

CREATE POLICY "class_members_select"
    ON class_members FOR SELECT
    USING (
        is_class_creator(class_id)
        OR user_id = auth.uid()
        OR status = 'approved'
    );

CREATE POLICY "class_members_insert"
    ON class_members FOR INSERT
    WITH CHECK (
        is_authenticated()
        AND user_id = auth.uid()
        AND status = 'pending'
        AND NOT is_class_member(class_id)
    );

CREATE POLICY "class_members_update"
    ON class_members FOR UPDATE
    USING (
        is_class_creator(class_id)
        OR user_id = auth.uid()
    )
    WITH CHECK (
        is_class_creator(class_id)
        OR user_id = auth.uid()
    );

CREATE POLICY "class_members_delete"
    ON class_members FOR DELETE
    USING (
        is_class_creator(class_id)
        OR user_id = auth.uid()
    );

-- class_approval_requests 表策略
ALTER TABLE class_approval_requests ENABLE ROW LEVEL SECURITY;

DROP POLICY IF EXISTS "class_approval_requests_select" ON class_approval_requests CASCADE;
DROP POLICY IF EXISTS "class_approval_requests_insert" ON class_approval_requests CASCADE;
DROP POLICY IF EXISTS "class_approval_requests_update" ON class_approval_requests CASCADE;
DROP POLICY IF EXISTS "class_approval_requests_delete" ON class_approval_requests CASCADE;

CREATE POLICY "class_approval_requests_insert"
    ON class_approval_requests FOR INSERT
    WITH CHECK (
        auth.uid() = user_id
        AND status = 'pending'
    );

CREATE POLICY "class_approval_requests_select"
    ON class_approval_requests FOR SELECT
    USING (
        is_super_admin()
        OR user_id = auth.uid()
    );

CREATE POLICY "class_approval_requests_update"
    ON class_approval_requests FOR UPDATE
    USING (is_super_admin())
    WITH CHECK (is_super_admin());

CREATE POLICY "class_approval_requests_delete"
    ON class_approval_requests FOR DELETE
    USING (is_super_admin());
```
