建议的解决方向
由于本地 Node.js / TypeScript 环境可能有缓存或配置问题，建议以下方案：

方案一：在服务器上直接构建
跳过本地构建环境，直接在服务器上运行：


npm run build
npm start
这样可以避免本地环境问题。

方案二：清理本地构建缓存
如果需要在本地构建，尝试：

清理 .next 目录：


rm -rf .next
rm -rf node_modules/.cache
重启 TypeScript 服务器：

在 VS Code 中按 Ctrl+Shift+P，选择 "TypeScript: Restart TS Server"
删除 package-lock.json 后重新安装依赖：


rm package-lock.json
npm install
方案三：使用简化版本
如果方案一、二都失败，考虑使用更稳定的构建配置：

降级 TypeScript 版本
使用 next build 而不是 npm run build
临时禁用 ESLint/TypeScript 检查
